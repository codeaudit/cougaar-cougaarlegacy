Liaison Operation

As noted at the beginning, this tutorial assumes that the liaison capability has already been properly installed and configured.  Any use of the liaison capability currently requires the availability of CoABS Grid-based Jini services (to support the CoABSLiaisonDeputy class) and a JavaSpaces service (to support the LiaisonManager class and liaison administration in general).  The following instructions activate the necessary processes; they refer to batch files in the bin directory of the liaison capability distribution:

Starting the CoABS Grid

  • Run the script StartCoABS.bat. 
  • A command prompt window and a GUI window will open. 
  • In the GUI window click on the "Delete logs" button. 
  • Then click on the "Start Grid Daemons" button. 
  • Then select the "Daemon Output" tab and wait for a confirmation line to appear in the HTTP output pane. 
  • Select the "Grid Controls" tab and click on the "Start Logger Agent" button. 
  • Select the "Daemon Output" tab and wait for a second confirmation line to appear in the HTTP output pane. 
  • Select the "Grid Status" tab and fill in the field at the top with the DNS name of the machine the Grid is running on. 
  • Click on the "Find Agents" button. 
  • In a few second the display should change to show 2 registered agents; this confirms that the Grid is functional. 
  • You may shrink both the GUI window and the command prompt window.

Starting a JavaSpaces service

  • Run the script StartJiniTransactions.bat.
  • A command prompt window will open.
  • At the prompt, answer Y.
  • In a few seconds the window will disappear.
  • Run the script StartTransientJavaSpace.bat.
  • A command prompt window will open.
  • You may shrink this window.
  • Run the script StartAppWeb.bat on the host where the ALP society will be running.
  • A command prompt window will open.
  • You may shrink this window.
