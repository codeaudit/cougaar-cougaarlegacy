ALP/Cougaar Liaison Capability Tutorial

If you have read through the overview Javadoc, reviewed the presentation slides for the liaison capability, and browsed through some of the package and class Javadocs, you are ready to explore the mechanics of actually implementing useful liaisons between your ALP/Cougaar society and some external agents. This short tutorial is designed to lead you through the fundamental steps with example code illustrating the key concepts, and with additional discussion that explores alternatives and ramifications.

Note: the tutorial assumes that the liaison capability has already been properly installed and configured. Although it is not dependent upon the weather demonstration (including the Litton TASC AEF society and RETSINA InfoAgents), the liaison capability does require Jini and JavaSpaces services in order to function properly.  These constraints are discussed in the section on liaison operation.

Liaison PlugIns shows how to begin by providing an environment in which ALP/Cougaar PlugIns can identify themselves and perform interactions with external agents.

Liaison Deputies describes how to initialize communications with external agent societies

Basic Interactors shows how to construct the core components required to implement specialized interactions between an ALP/Cougaar agent and some external agent.

Interactions explores the concept of liaison interactions and the state-driven engine that drives all liaison activity.

Multithread Interactors explores the problem of building interactors that must be manipulated by multiple threads.

Timeouts shows how to set timeout limits at different levels within an interaction and how to deal with them when they occur.

Liaison Administration discusses the experimental JavaSpace-based facility for controlling liaison permissions and how to use or bypass it.

Interaction Factories describes how to deal with interactions that are initiated by external agents, and how to enable deputies to instantiate these interaction when needed without requiring repeated intervention by a PlugIn.

Liaison Operation covers the mechanics of getting liaisons running, including the Jini and JavaSpace services upon which they (currently) depend.
