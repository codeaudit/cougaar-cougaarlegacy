ALP External Agent Liaison Capability Release Notes

Current Version: 1.0.1

Latest Release Version: 1.0

Latest Release Date: 11/27/00

Summary

This is primarily a compatibility upgrade for the ALP external agent liaison capability.  It is now compatible with ALP/Cougaar mini-build version 7.0.0.

Version 1.0.1 changes from prior release (version 1.0)

New capabilities and improvements

  • All references to packages within the "alp.*" and "mil.darpa.log.alp.*" package trees were replaced with the appropriate "org.cougaar" package references.  This maintains compatibility with the released ALP/Cougaar mini-build version 7.0.0 as well as with the matching version of the Litton TASC AEF society.

  • A new RETSINA information agent (WCNForecastWeatherAgent) was implemented to extract weather forecast data from the Weather Channel web site.  A tool for scanning Weather Channel web pages to find particular weather examples was implemented.  The LiaisonTest object/application was modified to interact with this new agent, and the RETSINA interoperator scripts were modified to allow multiple interoperators to run at the same time.

Bug fixes

  • None.

Installation information

See the installation instructions.

Version 1.0 changes from prior release (version 0.9)

New capabilities and improvements

  • The ALP weather forecasting task was split into a weather requirement task (one per affected prepare-for-transport task) and a get-forecast task.  Get-forecast tasks are now multi-parent tasks that are a composition of multiple weather requirement tasks representing the same location and date.  The WeatherLiaisonPlugIn now updates get-forecast results from forecast data acquired via the Grid, and the weather trigger updates the itinerary of the affected prepare-for-transport tasks when the results flow back through the task compositions.  The weather trigger also saves the original itineraries for later display and comparison.  Defunct weather requirements (for past-current dates or for unknown locations) are correctly purged from the plan.

  • A new PSP exists (liaison/demo/WEATHER_IMPACT.PSP) that displays the results of weather forecast requests and their impact on prepare-for-transport tasks.

  • A software brake that regulated the speed of spawning new queries to the RETSINA information agent has been removed.

  • Portions of the Liaison Administration Tool GUI were implemented with multiple threads to improve apparent response times.

Bug fixes

  • Corrected a race condition which caused the Liaison Administration Tool GUI to fail to update liaison status properly under certain circumstances.
