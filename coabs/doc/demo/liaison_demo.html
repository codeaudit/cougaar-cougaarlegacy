ALP External Agent Liaison Capability Demo Instructions

Preparation

Running the ALP Liaison capability demo involves bringing up two other complex software frameworks besides ALP: the CoABS Grid and RETSINA.  All of the run-time preparation work is accomplished by scripts in the Liaison bin directory.  (The instructions assume you double-click on these scripts from an Explorer window).

Starting the CoABS Grid

  • Run the script StartCoABSGrid.bat. 
  • A command prompt window and a GUI window will open. 
  • In the GUI window click on the "Delete logs" button. 
  • Then click on the "Start Grid Daemons" button. 
  • Then select the "Daemon Output" tab and wait for a confirmation line to appear in the HTTP output pane. 
  • Select the "Grid Controls" tab and click on the "Start Logger Agent" button. 
  • Select the "Daemon Output" tab and wait for a second confirmation line to appear in the HTTP output pane. 
  • Select the "Grid Status" tab and fill in the field at the top with the DNS name of the machine the Grid is running on. 
  • Click on the "Find Agents" button. 
  • In a few second the display should change to show 2 registered agents; this confirms that the Grid is functional. 
  • You may shrink both the GUI window and the command prompt window.

Starting RETSINA

  • Run the script StartRETSINAANS.bat.
  • A command prompt window will open.  There will be some debug output, including a couple of warning messages about files not existing or not being readable; this is normal and my be ignored.
  • You may shrink the command prompt window.
  • Run the script StartRETSINAForecastWeatherAgent.bat.
  • A command prompt window and a GUI window will open.
  • The GUI window will show two error messages; this is normal.  You may shrink this window.
  • Wait for the command prompt window to show "Ready to receive objectives" and then you may shrink it.
  • Run the script StartRETSINAForecastInteroperator.bat.
  • A command prompt window will open.
  • Wait for the command prompt window to show a line that says "done." and then you may shrink it.  (Occasionally the interoperator doesn't quite flush out this message but it still seems to work OK).
  • Run the script StartRETSINAANSTest.bat.
  • A command prompt window will open.
  • At the ANS prompt, type list all.
  • Two agents should be listed (InterOp and CNNForecastWeatherAgent).  This verifies that the RETSINA ANS, Interoperator, and Weather Agent are all functional.
  • Type quit and the command prompt window will disappear.

Liaison demo preparation

If you are running a multi-machine demonstration, you must first start up some supporting RMI and Jini processes:

  • Run the script StartJiniRMID.bat on the LIAISON_JINI_HOST (this must be a different machine from the one hosting the CoABS Grid service).
  • A command prompt window will open.
  • At the prompt, answer Y.
  • You may shrink this window.
  • Run the script StartJiniWeb.bat on the LIAISON_JINI_HOST.
  • A command prompt window will open.
  • You may shrink this window.
  • Run the script StartJiniLookup.bat on the LIAISON_JINI_HOST.
  • A command prompt window will open and then disappear in a few seconds.

You are now ready to start the Liaison demo JavaSpace service and the HTTP service supporting download of liaison proxy objects upon request.

  • Run the script StartJiniTransactions.bat.
  • A command prompt window will open.
  • At the prompt, answer Y.
  • In a few seconds the window will disappear.
  • Run the script StartTransientJavaSpace.bat.
  • A command prompt window will open.
  • You may shrink this window.
  • Run the script StartAppWeb.bat on the host where the ALP Liaison demonstration society will be running.
  • A command prompt window will open.
  • You may shrink this window.

Liaison Demo

After all of the preceding setup has been performed, you may run the Liaison demo:

  • Open a command prompt window.
  • Change to the Liaison alp directory.
  • Run the script liaisondemo.bat.
  • As PlugIns load, a number of Java windows will open.
  • After all PlugIns have loaded (the text "Plugins Loaded" will appear in the command prompt window), click on the "Propagate OPlan 2001 Scenario" button in the JFACC window.
  • Then click on the "GetLogSupport 2001 Scenario" button.
  • In addition to the normal heartbeat and message characters (".", "+", and "-"), some debug output and error messages "unable to allocate AEF to SubSquadron," "ConsumerSpec was null," and "##GLM_ERROR##" may appear; these are OK (for demo purposes).
  • Eventually, the message "CoABSRegistryInteractor waiting for permission" will appear (and repeat periodically).
  • In another command prompt window, or directly from an Explorer window, run the script runTool.bat from the Liaison alp directory.
  • The ALP Liaison administration tool GUI will appear.
  • From the ALP Society menu choose "ALP_Society".
  • After the society appears in the left pane, choose "CoABSGrid" from the External Society menu.
  • Select and modify the ALP_Society society and Transport neighborhood (one at a time) from the left pane so that they delegate permission settings for their subordinates.
  • Select the GlobalAir agent in the left pane.
  • Select the Registry (LookupDiscovery) and CNNForecastWeatherAgent agents from the right pane (one at a time) and modify their permissions so that GlobalAir may inititiate interactions with them.
  • You may close the administration GUI window.
  • Some "[" and "]" characters will appear in the original command prompt window.  These indicate that the ALP society is interacting with the Grid and the RETSINA Weather Agent.
  • Start a browser and open the URL http://<ALP_host_name>:5555/$GlobalAir/liaison/demo/WEATHER_IMPACT.PSP.  You will see a display of weather forecasts, their expected impact, and the tasks they affect.  Selecting a task will show the details for that task's itinerary and how it will be affected by the forecast weather conditions.
